# 安全政策

## 支持的版本

目前我们仅为最新版本的 WebGIS Home 提供安全更新。

| 版本  | 支持状态  |
| ----- | --------- |
| 1.0.x | ✅ 支持   |
| < 1.0 | ❌ 不支持 |

## 报告漏洞

如果您发现本项目存在安全漏洞，请**不要**通过公开 Issue 报告。

相反，请发送电子邮件至 [18681693083@163.com](mailto:18681693083@163.com)，包含以下信息：

- 漏洞类型
- 完整的漏洞路径（包括依赖项版本）
- 逐步说明如何利用该漏洞
- 其他相关信息（例如概念验证代码、截图等）

我们将尽快回复您（通常在 48 小时内），并在修复后向您通报进展。

## 披露政策

我们遵循负责任的披露原则：

1. 您报告漏洞
2. 我们确认收到并调查问题
3. 我们修复漏洞并发布补丁
4. 我们公开披露漏洞详情（在用户有机会更新后）

## 安全最佳实践

### 对于用户

- 始终使用最新版本的 WebGIS Home
- 定期更新依赖项
- 不要在生产环境中暴露敏感配置信息
- 使用 HTTPS 部署应用

### 对于贡献者

- 不要在代码中硬编码密钥或令牌
- 使用环境变量管理敏感配置
- 审查第三方依赖的安全性
- 遵循 OWASP 安全指南
